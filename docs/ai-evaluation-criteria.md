# AI 기능 평가 기준

> **목적:** Phase 2 AI/ML 기능의 성능, 신뢰도, 공정성 평가 기준 정의  
> **적용 범위:** AI 결함 탐지, 균열 분석, 진단 의견, 민원 분류

---

## 1. AI 결함 탐지 (DEFECT_DETECTION)

### 1.1 모델 정보

| 항목 | 내용 |
|------|------|
| 알고리즘 | Mask R-CNN (인스턴스 세그멘테이션) |
| 대상 | CRACK, LEAK, SPALLING, CORROSION, EFFLORESCENCE |
| 입력 | RGB 이미지 (드론/모바일 촬영) |
| 출력 | BBox 좌표, 신뢰도(0~1), 결함 타입, 심각도 추정 |

### 1.2 성능 평가 기준

| 지표 | 최소 기준 | 목표 기준 |
|------|-----------|-----------|
| Precision (정밀도) | ≥ 0.80 | ≥ 0.90 |
| Recall (재현율) | ≥ 0.75 | ≥ 0.85 |
| mAP@0.5 | ≥ 0.70 | ≥ 0.80 |
| 오탐률(FPR) | ≤ 0.15 | ≤ 0.08 |
| 미탐률(FNR) | ≤ 0.20 | ≤ 0.12 |

### 1.3 신뢰도 등급 처리 기준

| 신뢰도 범위 | 등급 | 처리 방식 |
|-------------|------|-----------|
| ≥ 0.90 | AUTO_ACCEPT | AI_AUTO_ACCEPT 플래그 ON 시 자동 확정 |
| 0.80 ~ 0.89 | REQUIRES_REVIEW | 검토자 확인 버튼 클릭 필요 |
| < 0.80 | MANUAL_REQUIRED | 수동 입력 유도, AI 결과 참고용 |

### 1.4 평가 데이터셋

- **검증 셋:** 건물 외벽 이미지 ≥ 500장 (균열 200, 누수 100, 기타 200)
- **라벨링 기준:** KCS 14 20 10 (콘크리트 균열 보수 기준) 준용
- **평가 주기:** 분기별 1회 재평가, 모델 업데이트 시 즉시 평가

## 2. 균열 분석 (CRACK_ANALYSIS)

### 2.1 모델 정보

| 항목 | 내용 |
|------|------|
| 알고리즘 | OpenCV.js WASM + Y-MaskNet (드론 정밀) |
| 특허 | 10-2398241 (비접촉 균열 측정 기술) |
| 출력 | 균열 폭(mm), 길이(mm), 깊이 추정(mm), ROI 이미지, 마스크 이미지 |

### 2.2 측정 정확도 기준

| 지표 | 최소 기준 | 목표 기준 |
|------|-----------|-----------|
| 폭 측정 오차 | ≤ ±0.3mm | ≤ ±0.1mm |
| 길이 측정 오차 | ≤ ±5% | ≤ ±2% |
| 측정 재현성(동일 이미지 3회) | CV ≤ 5% | CV ≤ 2% |
| KCS 기준 분류 일치율 | ≥ 85% | ≥ 92% |

### 2.3 KCS 기준 연계

| 균열 폭 | KCS 판정 | 시스템 처리 |
|---------|----------|------------|
| < 0.1mm | 경미 — 관찰 | severity = LOW |
| 0.1 ~ 0.3mm | 경계 — 모니터링 | severity = MEDIUM |
| 0.3 ~ 1.0mm | 주의 — 보수 검토 | severity = HIGH + alert |
| > 1.0mm | 긴급 — 즉시 조치 | severity = CRITICAL + alert |

## 3. AI 진단 의견 (DIAGNOSIS_OPINION)

### 3.1 모델 정보

| 항목 | 내용 |
|------|------|
| 모델 | Claude claude-sonnet-4-6 (Anthropic) |
| 입력 | 결함 목록, 균열 측정값, 센서 데이터, 민원 현황 |
| 출력 | 요약(1문장), 기술 의견(마크다운), 긴급도, 우선순위 점수 |

### 3.2 품질 평가 기준

| 평가 항목 | 기준 |
|-----------|------|
| 전문용어 사용 정확도 | 구조공학 용어 오용 없음 (검토자 검증) |
| KCS 기준 참조 여부 | 관련 기준 1개 이상 명시 |
| 긴급도 분류 일치율 | 전문가 판정과 ≥ 80% 일치 |
| 초안 승인율 | ≥ 70% (rejection 사유 기록 필수) |
| 응답 소요시간 | ≤ 15초 (p95) |

### 3.3 Human-in-the-loop 요건

- AI 의견은 **항상 DRAFT 상태**로 생성
- **REVIEWER 권한** 보유자만 APPROVED 상태로 변경 가능
- 승인/반려 이력 **영구 보존** (감사 추적)
- 반려 시 **이유 필수 입력** (모델 개선 피드백)

## 4. 민원 AI 분류 (COMPLAINT_TRIAGE)

| 항목 | 내용 |
|------|------|
| 모델 | Claude claude-sonnet-4-6 (Anthropic) |
| 입력 | 민원 제목, 내용, 첨부 이미지(선택) |
| 출력 | category, urgencyScore(0~100), routingSuggestions, classificationReason |

### 4.2 분류 성능 기준

| 지표 | 최소 기준 | 목표 기준 |
|------|-----------|-----------|
| 카테고리 분류 정확도 | ≥ 0.80 | ≥ 0.90 |
| 긴급도 점수 상관계수 | ≥ 0.70 (전문가 평가 대비) | ≥ 0.85 |
| 배정 추천 적중률 | ≥ 0.75 | ≥ 0.85 |
| 처리 속도 | ≤ 10초 (p95) | ≤ 5초 (p95) |

### 4.3 카테고리 정의 및 SLA

| 카테고리 | 정의 | 기본 SLA |
|----------|------|----------|
| FACILITY | 시설물 결함 (균열, 누수, 파손) | 48시간 |
| SAFETY | 안전 위협 (낙하 위험, 화재 위험) | 4시간 |
| NOISE | 층간 소음, 기계 소음 | 72시간 |
| SANITATION | 위생 문제 (쓰레기, 악취) | 48시간 |
| PARKING | 주차 분쟁, 불법 주차 | 24시간 |
| ELEVATOR | 엘리베이터 고장/이상 | 4시간 |

## 5. 모델 신뢰도 관리

### 5.1 드리프트 감지 기준

| 모니터링 지표 | 경고 임계치 | 조치 |
|---------------|-------------|------|
| 월별 거부율 증가 | +15%p 이상 | 재평가 트리거 |
| 평균 신뢰도 하락 | -0.05 이상 | 모델 점검 |
| 오탐 신고 건수 | 월 10건 이상 | 즉시 검토 |

### 5.2 평가 주기

| 평가 유형 | 주기 | 담당자 |
|-----------|------|--------|
| 자동 지표 모니터링 | 실시간 | 시스템 |
| 정기 성능 평가 | 분기 1회 | AI 운영팀 |
| 전문가 샘플 검토 | 월 1회 (50건) | 구조공학 전문가 |
| 모델 재훈련 결정 | 필요 시 | AI 운영팀 + 관리자 |

## 6. 설명가능 AI (XAI) 기준

AX-SPRINT 연구과제 요건에 따라 AI 판단 근거를 제공한다.

| 기능 | 설명 요소 | 구현 |
|------|-----------|------|
| 결함 탐지 | BBox 시각화 + 탐지 타입 | 이미지 오버레이 |
| 균열 분석 | ROI 이미지 + 측정값 + KCS 기준 참조 | crack-analysis-result.roiImageKey |
| 진단 의견 | technicalOpinionDraft (상세 근거 포함) | 마크다운 전문 |
| 민원 분류 | classificationReason + keywordMatches | 텍스트 설명 |
| 위험도 계산 | subScores 5개 항목 + 가중치 | 차트 시각화 |
