## 1. 본 제품이 AI 응용제품인 이유

**단순 정보시스템(MIS)과의 차이점:**

| 구분 | 단순 정보시스템 | 에이톰-AX (AI 응용제품) |
|------|--------------|----------------------|
| 데이터 처리 | 입력된 데이터를 저장·조회 | AI가 데이터를 해석하고 의사결정 추천 |
| 의사결정 | 사람이 모든 판단 | AI가 사전 판단 → 사람이 최종 승인 |
| 업무 트리거 | 수동 입력 필요 | 이상 감지 시 자동 알림·작업지시 생성 |
| 사용 목적 | 기록·이력 관리 | 사고 예방·비용 절감·처리 자동화 |
| AI 결과 가시성 | 없음 | confidence score, SHAP 근거, Before/After KPI 화면에 직접 표시 |

**에이톰-AX의 AI 특성 3가지:**
1. **실시간 AI 추론**: KoBERT 민원 분류, Y-MaskNet 결함 탐지, IsolationForest 이상 감지가 운영 중 실행
2. **Human-in-the-loop**: AI 신뢰도 95% 미만은 사람이 검토·승인하는 구조
3. **AI 결과 근거 설명(XAI)**: SHAP 7개 지표로 위험 점수 산출 이유를 화면에 표시

## 3. Human-in-the-loop 구조 설명

에이톰-AX는 AI 결과를 맹목적으로 실행하지 않습니다.  
**AI 신뢰도 → 처리 경로 자동 분기** 구조를 통해 안전성과 효율성을 동시에 확보합니다.

```
[AI 추론 결과]
      │
      ├── confidence ≥ 95% → [자동 처리] (담당자 배정, 작업지시 생성)
      │
      ├── 80% ≤ confidence < 95% → [검토 대기] → AI 검토 수신함 → 담당자 확인 후 승인
      │
      └── confidence < 80% → [수동 처리 필수] → 담당자가 처음부터 직접 처리
```

**화면에서 확인 가능한 Human-in-the-loop 요소:**
- `AI 검토 수신함 (/ai-inbox)`: 검토 대기 항목 목록 + APPROVE / REJECT / MODIFY 인라인 액션
- `AI 근거 패널 (XAI-SHAP)`: 위험 점수 산출 이유 7개 지표 + 내러티브
- `KoBERT 시뮬레이터 (/complaints/triage)`: 민원 텍스트 → 분류 확률 분포 + 신뢰도 등급 실시간 표시
- `AI 결함 탐지 검토 (/ai-detections)`: 탐지 후보 → 엔지니어 승인 → 결함 등록 전환

## 5. 사업계획서 핵심 메시지

> **에이톰-AX는 "정보를 저장하는 시스템"이 아니라 "AI가 판단하고 사람이 승인하는 운영 자동화 플랫폼"입니다.**

1. **AI가 먼저 본다**: 드론·IoT·민원 텍스트를 AI가 실시간 분석하여 결과와 근거를 화면에 표시
2. **사람이 최종 승인**: 신뢰도 기반 자동/검토/수동 분기로 오작동 리스크를 구조적으로 차단
3. **효과를 숫자로 증명**: Before/After KPI 화면(`/ai-performance`)에서 ROI 340%, 절감 7,350만원 직접 확인
4. **LH 전국 확장 시나리오**: SH → LH 확장 시 24만 세대 → 연 절감 20.6억원 추산 (Vision 2030 페이지)
