# Phase 2 파일럿 패키지

> **목적:** 실증 기관(공공임대주택 관리 기관)을 위한 Phase 2 기술 소개 및 파일럿 운영 지침  
> **대상 기관:** LH, SH, 지방 도시공사, 민간 임대 관리 법인

---

## 1. AX 플랫폼 Phase 2 개요

### 1.1 플랫폼 개요

AX(AI eXplainable) 공공임대주택 플랫폼은 AI·드론·IoT·RPA를 결합하여 시설물 유지관리의 **10단계 자동화**를 구현하는 국가R&D(AX-SPRINT) 과제의 핵심 산출물입니다.

**Phase 2 핵심 가치:**

| 가치 | 내용 |
|------|------|
| 비용 절감 | 드론 AI 탐지로 외벽 점검 인력 60% 절감 |
| 사고 예방 | 센서 + AI 위험도 예측으로 사전 정비 |
| 민원 처리 속도 | AI 자동 분류로 배정 시간 70% 단축 |
| 행정 자동화 | RPA로 반복 업무 80~100% 자동화 |
| 설명가능 AI | 모든 AI 판단에 근거·이력 제공 (감사 추적) |

### 1.2 Phase 2 주요 기능

| # | 기능 | 기술 | 자동화율 |
|---|------|------|----------|
| 1 | 드론 영상 자동 결함 탐지 | Mask R-CNN, Y-MaskNet | 자동 탐지 → 검토 확정 |
| 2 | 균열 폭 정밀 측정 | OpenCV WASM (특허 10-2398241) | 자동 측정 |
| 3 | AI 진단 의견 초안 생성 | Claude API (LLM) | 초안 생성 → 전문가 승인 |
| 4 | 민원 AI 자동 분류 | Claude API (LLM) | 분류·배정 추천 → 담당자 확정 |
| 5 | IoT 센서 연동 경보 | 7종 센서 + 임계치 경보 | 자동 경보 |
| 6 | 업무 자동화 규칙 | 날짜/상태/임계치 기반 RPA | 80~100% |
| 7 | 위험도 예측·유지보수 권장 | 다중 가중치 모델 | 자동 산출 → 담당자 승인 |

## 2. 파일럿 운영 조건

### 2.1 최소 하드웨어 요건

| 구성요소 | 최소 사양 |
|----------|-----------|
| 서버 (API + DB) | CPU 4코어, RAM 8GB, SSD 100GB |
| AI 워커 서버 | CPU 8코어, RAM 16GB (GPU는 선택) |
| 네트워크 | 업로드 50Mbps 이상 |
| 드론 | DJI Mavic 3 Enterprise 이상 (4K 영상) |
| IoT 센서 | REST API 또는 MQTT 지원 센서 |

### 2.2 소프트웨어 요건

| 항목 | 버전 |
|------|------|
| Node.js | 24.x |
| Docker | 24.x |
| CouchDB | 3.3.x |
| Redis | 7.x |
| MinIO | RELEASE.2024 이상 |

### 2.3 파일럿 기간 권장 구성

```
1주: 환경 구축 + 데이터 이관 + 사용자 교육
2~4주: 실 데이터 입력 + AI 기능 검증
5~8주: 전체 워크플로우 운영 + 성과 측정
```

## 3. 파일럿 성과 측정 지표

### 3.1 운영 효율성 KPI

| 지표 | 측정 방법 | 목표값 |
|------|-----------|--------|
| 점검 소요 시간 | 드론 점검 vs. 기존 인력 점검 | -60% |
| 결함 탐지율 | AI 탐지 / 전문가 검토 확정 | ≥ 85% |
| 민원 배정 시간 | 접수 → 담당자 배정까지 | ≤ 1시간 |
| 민원 SLA 준수율 | 카테고리별 기한 내 처리 | ≥ 90% |
| 자동화 실행 성공률 | COMPLETED / 전체 실행 | ≥ 95% |

### 3.2 AI 품질 KPI

| 지표 | 측정 방법 | 목표값 |
|------|-----------|--------|
| 결함 탐지 정밀도 | 승인건 / (승인건+거부건) | ≥ 80% |
| 민원 분류 정확도 | 담당자 수동 확인 대비 | ≥ 80% |
| 진단 의견 승인율 | APPROVED / 전체 생성 | ≥ 70% |

## 4. 파일럿 지원 패키지

### 4.1 기술 지원

- **온보딩:** 설치 가이드 + 1일 현장 교육
- **운영 지원:** 파일럿 기간 중 이메일/채팅 기술 지원
- **트러블슈팅:** ops-runbook-phase2.md 제공

### 4.2 교육 자료

| 자료 | 대상 |
|------|------|
| 관리자 사용 가이드 | ORG_ADMIN |
| 점검자 앱 가이드 | INSPECTOR |
| 민원 담당자 가이드 | COMPLAINT_MGR |
| 데모 시나리오 영상 | 모든 사용자 |

### 4.3 데이터 초기 세팅 지원

```bash
# 파일럿 기관용 시드 실행
yarn workspace @ax/api seed:master

# 기관 단지 정보 일괄 등록 (CSV 가져오기)
# Admin Web > 조직 설정 > 단지 가져오기
```

## 5. 파일럿 제약 사항 및 주의

| 항목 | 내용 |
|------|------|
| AI 자동 확정 | 기본값 OFF — 모든 AI 결과는 검토 후 확정 |
| 이메일/SMS 발송 | RPA_DRY_RUN = true (파일럿 중 실발송 안 함) |
| 외부 API 연동 | HIS, 공공 연동 미구현 (Phase 3 예정) |
| 모바일 앱 | iOS/Android 미배포 (파일럿 중 웹 UI만 사용) |
| SLA | 파일럿 환경은 운영 SLA 미보장 |
