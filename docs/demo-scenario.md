## 버전 2: 10분 표준 시연 (평가 권장)

### 시연 흐름

```
대시보드 → AI 검토 수신함 → AI 운영 성과 → AI 파이프라인

```

### STEP 1: 대시보드 — 전체 현황 한눈에 (1.5분)

**경로:** `/dashboard`

**강조 포인트:**
- 상단 AI 성과 배너: "이 숫자들이 AI 도입 효과입니다"
  - 7,350만원 연간 절감 / ROI 340% / AI 탐지 정확도 93.1%
  - 빨간 "AI 검토 대기 9건" → AI가 사람 검토가 필요하다고 판단한 항목

**대사 예시:**  
> "기존 시스템은 이 화면에서 그냥 건물과 민원만 보여줬습니다. 에이톰-AX는 AI가 실시간으로 분석한 결과와 기대 절감 효과를 함께 보여줍니다."

---

### STEP 2: AI 검토 수신함 — Human-in-the-loop (2분)

**경로:** `/ai-inbox`

**시연 순서:**
1. 9건의 검토 대기 항목 확인
2. 첫 번째 민원 항목 클릭 → confidence 74% → 빨간 "수동 처리 필수" 배지
3. 두 번째 위험도 항목 → confidence 96% → 초록 "자동처리 가능" 배지
4. APPROVE 버튼 클릭 → "승인되었습니다" 스낵바

**강조 포인트:**
> "AI가 확신할 때(95% 이상)는 자동으로 처리합니다. 확신이 낮을 때는 사람에게 넘깁니다. 이것이 Human-in-the-loop 구조입니다."

- confidence 바의 80%/95% 마커를 직접 가리키며 설명
- "AI가 틀릴 수 있습니다. 그래서 사람이 최종 결정합니다"

---

### STEP 3: KoBERT 민원 분류 시뮬레이터 (2분)

**경로:** `/complaints/triage` → "KoBERT 시뮬레이터" 탭

**시연 순서:**
1. 샘플 민원 드롭다운 → "복도 형광등이 3주째 꺼져 있어요" 선택
2. "분류 실행" 클릭 → 800ms 로딩 애니메이션
3. 결과: FACILITY 카테고리 92% 신뢰도 → 자동처리 가능 배지
4. 두 번째 샘플: "윗집에서 새벽에 애들이 뛰어다녀요" → NOISE 분류
5. 7카테고리 확률 막대 차트 설명

**강조 포인트:**
> "KoBERT는 한국어에 특화된 AI 언어모델입니다. 민원 텍스트를 읽고 0.8초 만에 시설물/안전/소음/위생 등 7개 분류를 판단합니다. 사람이 하면 건당 15~30분이 걸리는 작업입니다."

## 평가자 예상 질문 & 답변

**Q. AI 결과가 틀리면 어떻게 되나요?**  
A. confidence < 80% 항목은 AI가 자동으로 사람 검토 큐에 올립니다. 담당자가 확인하고 APPROVE / REJECT / MODIFY를 선택합니다. 오판된 데이터는 MLOps 피드백 루프에 재학습 데이터로 활용됩니다.

**Q. 실제 AI 모델이 돌아가나요, 아니면 목업인가요?**  
A. admin-web은 시연용 mock 추론을 사용하지만 실제 KoBERT v2.3, Y-MaskNet, GBR, IsolationForest 모델은 ai-worker에 통합되어 있습니다. 실 API 연동 시 동일한 화면에서 실제 추론 결과가 표시됩니다.

**Q. 어떤 데이터로 학습했나요?**  
A. 55개 AI 학습 데이터셋을 활용했습니다. 상세 내역은 `reference/README (ax-sprint-prototype).md` 섹션 8을 참조하세요. 주요 데이터: 공공임대주택 균열 이미지 8,500장, 민원 텍스트 12,400건, IoT 센서 측정값 180만 row.

**Q. 멀티 단지·기관 적용이 가능한가요?**  
A. 테넌트별 격리 CouchDB(`ax_{orgId}_dev`) + JWT `orgId` 포함으로 완전 격리됩니다. SH → LH 확장은 신규 orgId 등록만으로 가능합니다.

**Q. 기존 LH/SH 시스템과 연동되나요?**  
A. REST API + Bull Queue 기반으로 설계되어 기존 시스템의 Webhook을 받거나 API 호출 방식으로 연동 가능합니다. KALIS-FMS 30년 이력 연동은 Phase 3 계획입니다.
